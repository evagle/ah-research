# Moat Framework — 本项目的护城河分析与打标准绳

本文件为本项目 value-profile skill 运行时读的操作级参考。关心"怎么做"，不关心"理论出处"。

---

## 1. 5类护城河分类法

每个护城河必须归入以下5类中的一类或几类。**"什么都有" ≈ "什么都没有"**, 若分不清请重做。

| 类别 | 定义 | 判定线 | 识别信号 |
|---|---|---|---|
| **大** | 在整个行业利润池里占比够大, 规模本身就是壁垒 | 行业利润占比 ≥ 20%, 或 营收为 #2的 ≥ 1.5× | 行业集中度 CR3/CR5、龙头相对份额 |
| **准** | 未来可能变成宽护城河, 但证据尚不充分 | 趋势指标 (份额/毛利率/用户数) 连续3年单向变化 | 趋势线, 需带时间窗口 |
| **强** | 用户心智/品牌/口味/习惯形成的壁垒, 换走要付心理成本 | 连续5年毛利率 ≥ 40% 且无下滑, 提价不掉量; 毛利率 ≥ 60% 且10年无下滑 = 顶级强护城河 (茅台/可乐级) | 提价事件响应, 复购率, NPS |
| **省** | 资本开支低/库存不贬值/存货周转快/折旧压力小, 赚的钱能留下来 | CFO/NI ≥ 80%, 维持性 CapEx/折旧 ≤ 1.2×; 10y 累计 Capex / 10y 累计 NI < 25% = 宽; 25-50% 中; > 50% 弱 (硬阈值, 见 §3.5) | 自由现金流转换率 |
| **专** | 技术/工艺/专利/监管门槛形成的壁垒, 别人做不了 | 研发投入占比 ≥ 行业均值1.5×, 或垄断牌照 | 专利引用数, 认证资质 |

**"准"类上限为中**: 只要一条护城河打 "准", 其最终标签最多是 "中", 不能升到 "宽"。因为 "准" 本身就是 "尚待证明"。

多类叠加允许（e.g. 高端消费品龙头 可能同时属 强+大+省）但每类都要独立论证。

## 2. 宽/中 / 窄/弱 四档标签

分析完5类后, 对整个公司的护城河给一个综合标签。**必须引用具体数字, 不接受泛泛而谈**。

| 标签 | 定性 | 硬指标 (任选2满足) |
|---|---|---|
| **宽** | 多类叠加 + 长期验证 | ≥ 10y 持续提价权 / 10y avg ROE ≥ 24% 且所有年份 ≥ 15% / ≥ 10y 毛利率 ≥ 40% 且无下滑 / 10y 累计 Capex/NI < 25% / 净利率 ≥ 20% 持续/行业 #1且领先 #2超50% 份额 |
| **中** | 单类清晰, 或多类不够长 | 5-10y 上述指标 / 10y avg ROE 15-24% / 10y 累计 Capex/NI 25-50% / 行业 #1-3 / 提价能力受限但份额稳定 |
| **窄** | 有护城河迹象但未跨周期验证 | 3-5y 指标持续 / 10y avg ROE 10-15% / 利基市场领导者/面临明显替代威胁 |
| **弱** | 指标参差/周期性强/易被复制 | < 3y 数据 / 10y avg ROE < 10% 或负年份 / 10y 累计 Capex/NI > 50% / 份额波动 ≥ 5pp / 技术或政策变更就可能洗牌 |

**无护城河 = 不要**: 若评估为 "弱", 直接在 Part 0标注 "定性研究 only", Step 6不输出估值数字。

## 3. 2项可证伪检验 — 必选2项

护城河判定必须带 **可证伪的定量检验**, 从以下4个中任选2个, 每个给出具体 pass/fail 阈值。

### 3.1提价测试
- **问**: 最近5年有几次提价? 提价后销量、份额、毛利率如何变化?
- **pass**: 提价后2 quarter 内 销量 < 5% 下降, 毛利率升 ≥ 100bp
- **fail**: 销量下滑 ≥ 10% 或毛利率被费用吞掉

### 3.2对手测试
- **问**: 最近3年主要竞争对手做了什么? 份额/价格/新品有无蚕食?
- **pass**: 竞对份额提升 ≤ 1pp / 年, 或竞对毛利率持续低于本公司 ≥ 5pp
- **fail**: 竞对以低价/补贴抢下 ≥ 3pp 份额

### 3.3切换成本测试
- **问**: 用户换走一次要付多少代价 (金钱/时间/心理/合规)?
- **pass**: 换走需重新培训/重走审批/数据迁移成本 > 年使用成本20%
- **fail**: 换走只需重新下单

### 3.4 ROE 路标测试
- **问**: 过去5-10年 ROE 的最低值和平均值
- **顶级 pass**（宽护城河标志）: 10y avg ≥ 24%, 所有年份 ≥ 15%, 趋势稳定/上行
- **pass**: 5y min ROE ≥ 12%, 10y avg ≥ 15%, 无负年
- **fail**: 周期中有负 ROE 或 < 5% 的年份
- **注**: ROE 由外部杠杆堆起（e.g. 长期负债/净资产 > 1）不算真 ROE, 必须同时满足 §3.5资本消耗测试 才算护城河信号

### 3.5资本消耗测试
- **问**: 过去10年累计 Capex（资本开支）占累计净利润的比例。资本消耗反映赚的钱是否真的归股东, 还是被"买新设备/补折旧"吃掉。
- **顶级 pass**（宽护城河标志）: 10y 累计 Capex / 10y 累计 NI **< 25%**（可乐 / Moody's / 茅台级）
- **pass**: 25% ≤ 10y Capex/NI ≤ 50%
- **fail**: 10y Capex/NI > 50% → 不是真赚钱, 是 "为生存而花钱"; 哪怕报表利润漂亮, 护城河定性不超过 "窄"
- **辅助指标**: 折旧/毛利 —— 护城河公司6-8%（轻资产）, commodity 公司常 > 22%; 折旧占比异常高常伴随高 Capex, 二者互为交叉检验

### 选型指导
- 消费品/品牌 → 提价 + ROE 路标 + 资本消耗
- B2B / 企业服务 → 切换成本 + 对手
- 周期行业 → ROE 路标（看穿越周期）+ 资本消耗 + 对手
- 高研发门槛 → 切换成本 + ROE 路标 + 资本消耗
- **通则**: §3.5资本消耗 是**必测项**（无论什么行业都跑）, §3.1-§3.4再任选1项搭配 —— 合计 "任选2" 变 "必 §3.5 + 任选1"

## 4. 跨年定量追溯 — 强制5年

每份 profile 的 §3护城河段必须给出以下数据表:

| 指标 | t-4 | t-3 | t-2 | t-1 | t | 趋势 |
|---|---|---|---|---|---|---|
| 毛利率 | | | | | | |
| 净利率 | | | | | | |
| ROE | | | | | | |
| CFO/NI | | | | | | |
| Capex/NI | | | | | | |
| 折旧/毛利 | | | | | | |

**规则**:
- 每个单元格必须带 **年报页码** 引用, 否则视为未完成。
- 趋势列允许 ↑ / ↓ / → / U / ∩ (U 为先降后升, ∩ 反之)。
- 任一指标连续2年下滑 ≥ 300bp → 护城河打问号, 必须在附录给出原因分析。
- **§3.5硬阈值**: 10y 累计 Capex / 10y 累计 NI > 50% → 护城河综合标签降一级（§2 "宽" → "中", "中" → "窄"）; 累计口径 > 单年口径, 避免单年异常值干扰。
- **折旧/毛利 > 22% 警戒**: 若3年以上维持 ≥ 22%, 标注 "高折旧压力 —— 疑似 commodity business 而非护城河"。

## 5. 悲观情景 — 不接受 "如果市场变差"

悲观情景必须具体, 禁止泛泛。以下4个维度至少覆盖2个:

- **技术替代**: 新技术/新材料/新产品切掉主营的具体路径。
- **偏好迁移**: 用户结构变化（老龄化/年轻化/消费降级/升级）怎样切换到替代品。
- **监管冲击**: 行业政策/价格管制/税收/反垄断/环保 的具体打击面。
- **对手反击**: 现有对手或新进者在 price / product / channel / talent 某一维度的 specific playbook。

每条场景必须给出: **触发事件 + 量化影响 + 预计窗口**（e.g. "若 XX 技术在2-3年内成熟, 公司30% 营收受冲击"）。

## 6. 护城河分析报告结构

```
§3.1 定性判断（3-5 句, 必提 5 类之一或几类）
§3.2 来源因素（5 类各一段, 不适用的说 "本公司无此类护城河"）
§3.3 产品差异化（单独一段, 与 §3.2 的 "强" 类呼应但不重复）
§3.4 波特五力（强制 5 个维度, 每个给具体判定）
§3.5 2 项可证伪检验（§3 内节, 必出现）
§3.6 跨年定量追溯表（§3 内节, 必出现）
§3.7 悲观情景（§3 内节, 必出现）
§3.8 综合标签 + 置信度
```
