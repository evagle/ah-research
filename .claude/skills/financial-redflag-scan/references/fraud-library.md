## 1. 风险清单（简版 — 高危模式 10 项）

遇到以下任一模式, 立即降级为 "观察" 或直接剔除。每项是**阈值型**判定, 用于 29 项排雷的 quick overlay。

1. **商誉 / 净资产 > 30%** — 一次减值能让净资产腰斩
2. **其他应收款 / 净资产 > 10%** — 资金被挪用或输送的高发区
3. **在建工程 > 3 年不转固** — 要么工程失败, 要么规避折旧
4. **经营现金流 / 净利润 < 50% 连续 2 年** — 利润真实性风险, 可能是应收膨胀
5. **管理层道德风险** — 历史虚假陈述 / 处罚 / 关联交易非正常价 / 股东利益输送 → 直接大幅降级, 不再讨论生意好坏
6. **大存大贷** — 账上有几十亿现金 + 几十亿有息负债 + 仍在借款, 99% 现金造假
7. **生物资产 / 农林渔牧** — 獐子岛式可走动 / 会飞的资产, 审计困难
8. **应收账款增速 > 营收增速 ≥ 20pp 连续 2 年** — 销售乏力, 靠放账填营收
9. **10y 累计 Capex / 10y 累计 NI > 50%** — 赚的钱一半以上必须花在维持/扩张固定资产上, 不是真赚钱生意, 护城河定性不超过 "窄" (与 moat-framework.md §3.5 同一阈值)
10. **留存收益 10y 年化增速 < 8%, 或出现下行** — 留存收益 = 净利润 − 分红, 逐年累计代表复利再投资能力; 长期停滞或倒退 = 护城河瓦解 leading indicator, 比当年 ROE 下行出现更早

## 2. 三表勾稽公式（4 条必查）

勾稽是造假识别的**核心技术手段**: 三张表之间的数字必须互相对得上, 任一条不过 → 标 `需人工`。

### 2.1 真实营收
```
真实营收 ≈ 营收 + Δ合同负债 − Δ应收账款 − Δ应收票据 (商票部分)
```
若真实营收 << 报表营收, 利润被提前确认。

### 2.2 销售收现
```
销售收现比 = 销售商品 / 提供劳务收到的现金 / 营业收入 (含税)
```
健康值 ≥ 1.05（因为营收不含税, 收现含税）。< 1 连续 2 年, 应收账款或合同负债异常。

### 2.3 净利润 → 经营现金流桥
```
CFO = 净利润 + 折旧摊销 − Δ应收 − Δ存货 + Δ应付 + Δ合同负债 + 其他非现金
```
走一遍桥, 每一项都要有合理解释。不平的部分就是可疑。

### 2.4 维持性 CapEx 近似
```
维持性 CapEx ≈ 折旧摊销 × (1 + 通胀系数)
自由现金流 ≈ CFO − 维持性 CapEx
```
若 CFO > 总 CapEx 长期成立, 属 "省" 类生意。

## 3. 造假识别的 5 个维度（高频作假手法库）

不特指哪家公司, 列出高频作假手法。每家公司跑一遍以下 5 个维度, 列 "本公司是否沾边 + 页码引用 + 金额"。

- **收入端**: 关联方购销 / 突击发货 / 提前确认工程完工 / 放宽信用（填放账到应收）
- **成本端**: 在建工程停止转固（少折旧）/ 存货不跌价 / 研发资本化比例异常高
- **现金端**: 期末短暂借款补现金 / 受限资金混入非受限 / 理财产品分类为 "现金等价物"
- **利润端**: 卖资产 / 政府补助 / 公允价值变动 / 一次性所得税返还 凑利润
- **结构端**: 频繁并购制造商誉 → 后续慢性减值 / 子公司并表时点选择 / 关联方剥离亏损业务

## 4. 造假模式 叙述 库（A1 / A2 / A3）

§1 / §3 是**数字型**和**手法型**; 本节是**模式型** — 帮子 agent 理解**为什么**数字异常 = 真实造假信号。29 项计分触发任一项, 且可对号入座到 A1/A2/A3, 在报告"发现的风险 summary" 段落明确引用并加注 "模式 A<N>"。

### A1 客户塞货 (channel stuffing)
- **信号组合**: AR 增速 > 营收增速 ≥ 20pp **且** 存货增速 > 营收增速 ≥ 20pp **且** 应付账款 (AP) 增速趋缓 / 压缩
- **为什么**: 公司把货硬压给经销商, 确认营收; 经销商卖不动导致 AR 积压 + 自家存货同步膨胀; 同时供应商也知道销售疲软, 不再给信用期, AP 不增反降
- **典型场景**: 白酒 / 消费品渠道库存周期顶部; 集采前夕的医药公司

### A2 库存减值掩盖 (inventory obsolescence hiding)
- **信号组合**: 存货周转天数翻倍以上 **且** 毛利率保持稳定 **且** 存货跌价计提 / 存货 比值 10 年不升反降
- **为什么**: 存货占用资金翻倍 = 滞销; 正常情况毛利率应下降 (降价清库) 但却稳定 = 没做减值; 跌价计提比例反而走低 = 故意压缩减值以保利润
- **典型场景**: 周期股晚期 (钢铁 / 煤炭 / 猪); 电子 / 家电 technology refresh 前夕

### A3 供应商融资压力 (vendor squeeze)
- **信号组合**: 应付账款天数 (AP days) 从正常 60-90 天 被压缩到 30-40 天 **且** 营收加速 **且** 资产端现金仍在紧张
- **为什么**: 供应商担心公司经营不稳, 要求提前付款或缩短账期; 公司被迫动用现金流保供应链, 即便营收看起来在增长, 实际经营现金流承压
- **典型场景**: 高杠杆地产 / 扩张期互联网 / 大宗商品暴跌后的 commodity 公司; 也是破产前 2-4 季度的 early warning

**用法**:
- 上述 3 模式**不纳入 29 项计分**, 属于**深度解释层**。
- 主 agent 复核时: 29 项计分 ≥ 3 项且可聚类到同一 pattern A<N> = **系统性造假风险**, 报告结论升级为 "剔除"。

## 5. 与其他 reference 的分工

- **本文件 (`fraud-library.md`)**: 风险阈值 + 三表勾稽公式 + 5 维度作假手法 + 3 种模式 叙述 (造假判定)
